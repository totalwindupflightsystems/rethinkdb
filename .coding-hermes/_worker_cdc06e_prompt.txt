TASK: CDC-06e — Apply batch + reconnect/resync
Implement in src/clustering/replication_mailbox.cc and src/rdb_protocol/subscription.cc:

1. APPLY BATCH: subscription_applier_t::apply_batch() — write target changes + ledger entries in ONE serializer transaction. Each change_record_t is written to the target table, then a ledger entry (applied_change_t) is written recording the event_id + LSN.

2. DUPLICATE SUPPRESSION: Before applying any change_record_t, check if its event_id already exists in the apply ledger. If so, skip (idempotent). The ledger is a simple B-tree mapping event_id to applied LSN.

3. RECONNECT: On connection loss, the subscription reconnects and replays from the last confirmed LSN. The confirmed LSN is the highest contiguous LSN that has been applied AND acknowledged by the sink.

4. WAL GAP DETECTION: If the source WAL no longer has the requested starting LSN (retention advanced past it), transition subscription to RESYNC_REQUIRED state. This triggers a full re-snapshot.

5. RESYNC: When in RESYNC_REQUIRED, discard all partial apply state, request a fresh snapshot barrier from the source, and replay from the new barrier LSN.

6. IDEMPOTENCY: The same change_record_t applied twice must produce the same target state. Use event_id as the idempotency key.
